Research Program

AI-Driven Software Quality Engineering Research Agenda
This research program investigates how artificial intelligence, machine learning, and repository analytics can transform modern software testing and quality engineering. The program is structured as a sequence of research papers exploring predictive defect detection, intelligent automation, and autonomous testing systems.
Paper 1 — Repository Analytics & Defect Prediction ▶

Abstract

This research investigates how software repository analytics and machine learning techniques can identify defect-prone modules in large-scale software systems. Development metrics such as commit frequency, code churn, developer activity, and file evolution are used to train predictive models that support risk-based testing strategies.

Status: Completed — Manuscript Prepared (Not Submitted)

Paper 2 — AI-Driven Self-Healing Test Automation ▶

Abstract

This research proposes an AI-driven test automation framework capable of detecting and repairing failing automated test scripts caused by changes in user interface elements. Machine learning and DOM similarity analysis are used to dynamically recover broken locators and maintain test stability.

Status: In Progress

Paper 3 — Autonomous Regression Testing Systems ▶

Abstract

This research explores autonomous regression testing systems that automatically generate, prioritize, and execute test cases based on repository analytics and predictive defect models. The goal is to improve testing efficiency in continuous integration environments.

Status: Planned

Paper 4 — LLM-Based Test Case Generation ▶

Abstract

This research investigates how large language models can automatically generate software test cases from requirements, user stories, and functional specifications, enabling faster and more comprehensive testing processes.

Status: Planned

Paper 5 — Intelligent API Contract Testing ▶

Abstract

This research proposes an intelligent API contract testing framework capable of detecting compatibility issues between microservices by analyzing API specifications and runtime service interactions in distributed systems.

Status: Planned
