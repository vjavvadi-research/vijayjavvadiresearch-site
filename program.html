Research Program

AI-Driven Software Quality Engineering Research Agenda
This research program explores how artificial intelligence, machine learning, and repository analytics can transform software testing and quality engineering. The program focuses on predictive defect detection, intelligent automation, autonomous testing systems, and AI-driven software quality analytics.
AI-Driven Software Defect Prediction Using Repository Analytics ▶

This research investigates how repository analytics and machine learning can be used to predict defect-prone software modules before defects appear in production systems. Development activity metrics such as commit frequency, developer contributions, and code churn are analyzed to identify patterns associated with software defects.

The goal of this research direction is to develop predictive models that support risk-based testing strategies and allow software teams to focus testing efforts on high-risk components.

AI-Driven Self-Healing Test Automation Frameworks ▶

Modern test automation systems frequently break due to changes in user interface elements, dynamic identifiers, and evolving application structures. This research investigates self-healing automation frameworks capable of automatically adapting to UI changes without manual intervention.

Machine learning models, DOM similarity analysis, and intelligent locator recovery algorithms are explored to enable adaptive test automation systems that maintain long-term reliability across evolving applications.

Autonomous Regression Testing Systems ▶

Regression testing is essential for maintaining software quality during continuous development. However, executing large regression test suites can be costly and time-consuming.

This research explores autonomous regression testing systems that automatically generate, prioritize, and execute regression tests based on historical defect patterns and development activity.

The goal is to create self-optimizing testing systems that dynamically adapt regression strategies based on evolving software risk profiles.

Large Language Models for Automated Test Case Generation ▶

This research investigates how large language models (LLMs) can automatically generate test cases from system requirements, user stories, and software specifications.

Natural language processing techniques are used to convert textual requirements into structured automated test scenarios for user interface testing, API testing, and integration testing.

The objective is to reduce manual test design effort while improving test coverage across complex software systems.

Intelligent API Contract Testing for Microservices ▶

Modern microservice architectures rely heavily on API communication between distributed services. Ensuring compatibility between service providers and consumers is critical for system reliability.

This research investigates intelligent contract testing frameworks that automatically detect schema mismatches, contract drift, and integration risks across microservices.

Machine learning models are used to analyze API behavior and detect inconsistencies between expected and actual service responses.

AI-Powered Quality Analytics for Software Systems ▶

This research explores how artificial intelligence can analyze large volumes of software quality data including test results, defect reports, and development metrics.

The objective is to generate predictive insights that help software teams identify quality risks early in the development lifecycle and improve overall system reliability.

Machine Learning for Intelligent Test Prioritization ▶

Continuous integration pipelines often execute thousands of automated tests for every code change. Executing all tests can significantly increase build times.

This research explores machine learning techniques that predict which tests are most likely to fail based on historical test results and code changes.

The goal is to prioritize high-risk tests to accelerate software delivery while maintaining strong defect detection.

Autonomous UI Testing Using Computer Vision ▶

Traditional UI automation frameworks rely heavily on element locators that frequently break when application layouts change.

This research investigates computer vision techniques that allow automation frameworks to recognize UI components based on visual appearance rather than DOM identifiers.

The objective is to build more resilient UI automation systems capable of adapting to dynamic application interfaces.
