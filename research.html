Research Areas

Research domains in AI-Driven Software Quality Engineering
This research program explores how artificial intelligence, machine learning, and software repository analytics can transform modern software testing and quality engineering practices. The research focuses on predictive defect detection, intelligent automation systems, and autonomous testing frameworks designed to improve software reliability in large-scale software systems.

Repository Analytics & Defect Prediction

6-paper research series evaluating ML classifiers on 284,676 code instances across 5 open-source repositories. Random Forest achieves AUC 0.904. Risk-based testing captures 43% of defects from the top 10% of files.

AI-Driven Self-Healing Test Automation

Framework that recovers broken Selenium locators at runtime using DOM similarity analysis and ML-assisted ranking with 8 feature families. Mutation-based evaluation simulates realistic UI evolution scenarios.

LLM-Driven Test Case Generation

RAITG framework converting 312 requirements into executable tests via LLM + rule-based verification. Achieves 68% effort reduction, 94.1% requirement coverage, and 96.3% first-pass verification rate.

Cross-Repository Transfer Learning

Systematic leave-one-repository-out validation for defect prediction models. Multi-source training achieves F1=0.601 across diverse targets. Platt scaling calibration improves transfer by up to 0.079 F1.

Risk-Based CI/CD Testing Integration

Production deployment of Gradient Boosting model in FastAPI microservice with GitHub Actions integration. Achieves 4.3x improvement in defect detection efficiency via risk-ranked test execution.

AI Governance for Quality Engineering

Unified data governance layer with payload controls, content redaction, author privacy hashing, prompt versioning, and audit trails for responsible AI usage across all quality services.
