AI-Driven Software Quality Engineering Research
This page presents research papers, working manuscripts, and future scholarly work in artificial intelligence, software testing, intelligent automation, and software quality engineering.
Paper 1 — AI-Driven Software Defect Prediction Using Repository Analytics and Machine Learning: An Empirical Study on Large-Scale Open-Source Systems ▶

Abstract

Empirical evaluation of 6 ML classifiers (LR, DT, RF, GB, XGB, MLP) on 284,676 instances across 5 open-source repositories (Elasticsearch, Spring Boot, Hadoop, Kafka, Express). Random Forest achieves best F1=0.632, AUC=0.904. Risk-based testing shows top 10% of files capture 42.4% of defects (4.2x improvement).

Status: Completed

Paper 2 — Constructing a Multi-Repository Defect Prediction Dataset from Open-Source Software Systems: Methodology, Validation, and Benchmark Analysis ▶

Abstract

Large-scale dataset of 284,676 instances from 5 mature OSS projects with systematic methodology for repository selection, automated feature extraction, and defect labelling. 50-330x larger than existing benchmarks (PROMISE, AEEEM). Covers 10-year window (2013-2023).

Status: Completed

Paper 3 — Comparative Analysis of Machine Learning Classifiers for Software Defect Prediction: From Logistic Regression to Gradient Boosting and Neural Networks ▶

Abstract

Rigorous comparative evaluation of 6 classifiers with statistical significance testing (parametric + non-parametric), SMOTE impact analysis, and hyperparameter sensitivity studies. Ensemble methods significantly outperform single models. Cohen’s d=2.34 (p<0.001) for RF vs LR.

Status: Completed

Paper 4 — Feature Importance and Interpretability in Software Defect Prediction: A Multi-Method Analysis Using SHAP, Gini Importance, and Permutation Testing ▶

Abstract

Multi-method feature importance analysis across 6 model architectures. Code churn accounts for 28.5% of predictive signal. Kendall tau cross-model concordance tau=0.73-0.89. Permutation importance reveals Gini overestimates high-cardinality features by 20-30%.

Status: Completed

Paper 5 — Cross-Repository Transfer Learning in Software Defect Prediction: Leave-One-Out Evaluation and Domain Adaptation Strategies ▶

Abstract

Systematic leave-one-repository-out validation revealing defect rate mismatch as primary transfer failure driver. Multi-source training achieves F1=0.601 across diverse targets. Platt scaling calibration improves poorly-transferring scenarios by up to 0.079 F1.

Status: Completed

Paper 6 — Risk-Based Software Testing Using Repository Defect Prediction: Coverage Analysis, CI/CD Integration, and Operational Deployment ▶

Abstract

End-to-end operational deployment of Gradient Boosting model in production FastAPI microservice with GitHub Actions integration. Top 10% files capture 43.2% defects (4.3x improvement). Complete CI/CD pipeline integration with advisory vs gating rollout.

Status: Deployed in Production

Paper 7 — AI-Driven Self-Healing Web Test Automation Using DOM Similarity and Machine Learning ▶

Abstract

Framework that automatically recovers broken Selenium locators at runtime using DOM similarity analysis and ML-assisted ranking. Uses 8 feature families for weighted similarity scoring with hybrid heuristic + ML ranking. Mutation-based evaluation pipeline simulates realistic UI evolution.

Status: In Progress

Paper 8 — AI-Driven Test Case Generation from Natural-Language Requirements: A Prompt-Engineered, Rule-Verified Framework for Scalable Software Quality Engineering ▶

Abstract

RAITG framework converting 312 requirements across 3 domains into executable tests via LLM + deterministic rule-based verification. 68% effort reduction (184h to 58.9h). Requirement coverage improves from 71.2% to 94.1%. First-pass verification rate 96.3%.

Status: Completed / Production Service Deployed
